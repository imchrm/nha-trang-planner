## Логика системы весов (день = 4 единицы)

| duration | weight |
|---|---|
| `'Весь день'` | 4 |
| `'2 дня'` | 8 (особый кейс — занимает 2 слота) |
| `'Полдня'` | 2 |
| `'Вечер'` | 1 |
| `'2-3 часа'`, `'1.5 часа'` | 1 |

Карточка `AC5` (Далат 2 дня) с `weight=8` — единственная двухдневная. При попытке разместить в день, где следующий занят, показывается `BlockingAlertModal` с конкретными датами.

## Фазы поездки (date-adaptive)

```
getTripPhase(today):
  today < 30.05  → 'preparation'  (обратный отсчёт)
  today = 05.06  → 'birthday'     (конфетти, баннер, корал. фон)
  today > 09.06  → 'memories'     (режим воспоминаний)
  иначе          → 'active'       (маркер СЕГОДНЯ, приглушение прошлых дней)
```

Фаза определяется при монтировании, не реактивна к течению времени внутри сессии.

## Персистентность и шаринг

- **localStorage** — авто-сохранение при каждом изменении `schedule`
- **URL hash** — `#s=base64(JSON.stringify(schedule))`, генерируется в `ShareModal`
- **JSON файл** — экспорт/импорт через `<input type="file">`
- **QR** — `<img src="https://api.qrserver.com/...">` без библиотек
- **Конфликт** — при наличии и URL, и localStorage → `LoadConflictModal` (Вариант Б)
